# Explain Code 工作流

当用户要求解释代码时，按以下步骤执行：

## 步骤 1：读取代码
- 如果用户指定了文件路径，使用 `read_file` 读取
- 如果用户指定了函数名/类名，使用 `search_content` 定位后读取
- 如果用户直接粘贴了代码，直接分析

## 步骤 2：分析代码
从以下维度分析：

### 功能概述
- 这段代码做什么？（一句话总结）
- 属于哪个模块/功能？

### 逐行/逐段解释
- 将代码分为逻辑段
- 解释每段的功能和目的
- 标注关键变量和它们的用途

### 数据流
- 输入数据从哪里来
- 数据如何被处理/转换
- 输出数据到哪里去

### 控制流
- 主要的分支条件
- 循环的目的和终止条件
- 异常/错误处理逻辑

### 设计意图
- 为什么这样实现（而非其他方式）
- 有哪些设计模式或最佳实践
- 与其他模块的协作关系

## 步骤 3：输出解释

按以下结构输出：

### 概述
[一句话说明代码功能]

### 详细解释
[逐段解释代码逻辑]

### 关键点
- [要点1]
- [要点2]
- [要点3]

### 注意事项（如有）
- [需要注意的边界条件/副作用/性能问题]

## 注意事项
- 解释要面向用户的水平调整深度
- 如果代码有明显问题，在解释后委婉指出
- 使用类比帮助理解复杂概念
- 避免逐字翻译代码，要解释"为什么"而非仅仅"是什么"
