# Debug Error 工作流

当用户报告错误并要求调试时，按以下步骤执行：

## 步骤 1：收集错误信息
- 询问用户提供完整的错误信息（如果没有提供）
- 确认错误发生的时间、操作场景、环境信息
- 如果用户提供了错误日志或截图，仔细分析

## 步骤 2：定位错误源
- 根据错误信息中的文件路径和行号，读取相关代码
- 使用 `search_content` 工具搜索错误信息中的关键词
- 使用 `read_file` 读取报错文件的相关代码段
- 检查调用链，理解错误是如何传播的

## 步骤 3：分析根因
分析常见错误类型：
- **编译错误**：语法错误、类型不匹配、缺少导入
- **运行时错误**：空指针、数组越界、类型转换失败
- **逻辑错误**：条件判断错误、循环边界错误、状态管理错误
- **并发错误**：竞态条件、死锁、actor 隔离问题
- **网络错误**：超时、连接失败、API 格式错误

## 步骤 4：提供修复方案
- 明确指出根因（一句话总结）
- 提供具体的代码修改方案
- 解释为什么这个修改能解决问题
- 如果有多种修复方案，列出各方案的优缺点

## 步骤 5：验证修复
- 应用修复后，建议用户重新运行/编译验证
- 如果修复后仍有错误，重复步骤 1-4
- 如果错误已解决，确认修复完成

## 注意事项
- 不要只看错误信息的表面，要追踪到根本原因
- 修改代码前先理解原有逻辑，避免引入新问题
- 优先最小化改动，避免大规模重构
