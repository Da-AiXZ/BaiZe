# Test Generation 工作流

当用户要求生成测试时，按以下步骤执行：

## 步骤 1：分析待测代码
- 读取要测试的代码文件
- 理解函数/方法的输入、输出、副作用
- 识别边界条件和异常场景
- 确认测试框架（根据项目现有测试或询问用户）

## 步骤 2：设计测试用例
为每个函数/方法设计测试用例，覆盖：

### 正常路径
- 典型输入 → 预期输出
- 多组正常输入验证

### 边界条件
- 空输入（空数组、空字符串、nil）
- 单元素输入
- 最大值/最小值
- 首尾元素

### 异常路径
- 无效输入 → 预期错误/异常
- 权限不足 → 预期拒绝
- 资源不可用 → 预期降级

### 并发场景（如适用）
- 并发访问安全性
- 竞态条件

## 步骤 3：生成测试代码
- 根据项目使用的测试框架生成代码（Swift XCTest / pytest / Jest 等）
- 每个测试用例独立可运行，不依赖其他测试的执行顺序
- 使用有意义的测试函数名：`test_功能_条件_预期结果`
- 添加必要的 setUp/tearDown

## 步骤 4：组织测试文件
- 测试文件放在对应的测试目录中
- 文件命名遵循项目规范（如 `XXXTests.swift`）
- 如果项目已有测试文件，追加到现有文件

## 步骤 5：运行测试验证
- 执行生成的测试
- 确认所有测试通过
- 如果测试失败，分析是测试代码问题还是被测代码问题
- 修复测试代码直到全部通过

## 注意事项
- 测试应该快速且可重复
- 避免测试中的硬编码依赖（如网络、文件系统）
- 使用 mock/stub 隔离外部依赖
- 每个测试只验证一个行为
- 测试代码也需要保持可读性和可维护性
