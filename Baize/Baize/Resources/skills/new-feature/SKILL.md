# New Feature 工作流

当用户要求实现新功能时，按以下步骤执行：

## 步骤 1：需求分析
- 仔细理解用户的功能需求
- 确认功能范围和边界
- 询问不明确的需求细节
- 确认目标平台和技术栈

## 步骤 2：技术方案设计
- 分析现有代码结构，确定新功能的集成点
- 设计数据模型（新增的 struct/class/enum）
- 设计接口（新增的函数/方法/API）
- 考虑与现有功能的交互和兼容性
- 如果功能较复杂，先输出方案让用户确认

## 步骤 3：实现准备
- 使用 `list_directory` 了解项目结构
- 使用 `search_content` 查找相关已有代码
- 确认要创建的新文件和要修改的现有文件
- 确认编码规范（参考 BAIZE.md 或项目现有风格）

## 步骤 4：逐步实现
按以下顺序实现：

### 数据模型
- 先实现核心数据模型（struct/enum）
- 确保模型完整、类型安全

### 核心逻辑
- 实现业务逻辑函数
- 处理边界条件和错误情况
- 添加必要的输入验证

### 集成层
- 将新功能集成到现有代码中
- 更新依赖模块的调用
- 确保不破坏现有功能

### 配置和常量
- 添加必要的配置项
- 更新常量定义

## 步骤 5：验证
- 编译确认无错误
- 如果有测试，运行测试确认通过
- 建议用户手动验证功能
- 检查是否有遗漏的边界情况

## 步骤 6：收尾
- 清理临时代码和调试日志
- 确认代码风格一致性
- 如果需要，更新文档
- 总结实现的功能和注意事项

## 注意事项
- 每一步只做一件事，确认后再继续下一步
- 新代码应遵循项目现有的编码规范
- 优先复用已有代码，避免重复实现
- 如果功能涉及破坏性变更，提前告知用户影响范围
- 实现过程中如果发现需求不合理，及时与用户沟通
